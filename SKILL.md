# Video to Notes

跨平台视频转录 → 整理为结构化笔记的工作流。支持 YouTube / B站 / 小红书 / 抖音 四个平台，自动判断语言并生成对应的三部分或两部分笔记。

## 首次加载协议

加载本 skill 后，Agent 必须依次执行以下步骤，未完成前不得处理用户请求：

```
步骤 1 — 询问用户输出路径
  "你的笔记输出目录是？"（支持 Obsidian 或任意文件夹）
  将路径保存到记忆/配置中，后续直接使用

步骤 2 — 询问是否需要代理
  "访问 YouTube 需要代理，你有代理地址吗？"
  有 → 保存代理地址；无 → 跳过（YouTube 会失败）

步骤 3 — 安装依赖
  pip install -r requirements.txt

步骤 4 — 确认 ffmpeg
  ffmpeg -version | 如果没有 → 提示用户安装
    - Windows: winget install ffmpeg
    - macOS:   brew install ffmpeg
    - Linux:   apt install ffmpeg
```

> 注意：本 skill 的所有文件均使用 UTF-8 编码。Agent 在读取 .md 文件时必须使用 `encoding='utf-8'`，否则中文 Windows 默认的 GBK 编码会导致乱码或报错（无法解析 emoji 字符）。以 Python 为例：`open(path, 'r', encoding='utf-8')`。

### ⚠️ 笔记 Frontmatter 书写规范

每条笔记的 YAML 头部必须包含以下三个元数据字段（用于交付检查脚本的完整性和截断检测）：

```yaml
---
title: 视频标题
source: 小红书 / 抖音 / bilibili / YouTube
url: 视频链接
date: 2026-06-20
tags:
  - 转录
  - <平台名>
_duration: 757       # 视频时长（秒），必须从 yt-dlp 获取
_audio_mb: 11.6       # 下载的音频文件大小（MB）
_raw_chars: 4236      # 原始转录文本总字符数（Whisper 或 API 输出的纯文本长度）
---
```

三个字段的作用：
- `_duration` + `_audio_mb` → 检查音频下载是否被截断（三层校验：P3时间戳 → 128kbps估算 → 字数兜底）
- `_raw_chars` → 检查 Part 2 清理后是否删除过多内容。中文视频语速 ≥3.5 字/秒时保留率不得低于 35%（硬下限），正常语速 <45% 时报错。英文视频 <8% 时报错。

> 注意：以上阈值为 Whisper tiny 模型校准值。如使用 base/small 模型，清理保留率会更高。35% 硬下限确保低质量 tiny 转录不会被交付。

| 系统 | 状态 | 注意事项 |
|------|------|---------|
| Windows | ✅ 已验证 | 路径用 `C:\\Users\\...` 格式；yt-dlp 可能叫 `yt-dlp.exe` |
| macOS | ✅ 理论上兼容 | 路径用 `/Users/...`；需 `brew install ffmpeg` |
| Linux | ✅ 理论上兼容 | 路径用 `/home/...`；需 `apt install ffmpeg` |

### 跨平台关键差异

- **路径格式**：Python 的 `os.path` 会自动适配系统分隔符，无需手动处理
- **yt-dlp 调用**：脚本自动检测可执行文件名（yt-dlp.exe / yt-dlp）
- **ffmpeg**：Windows 需手动添加 PATH；macOS/Linux 通过包管理器安装
- **文件系统**：`B站/` `小红书/` 等中文目录名在三大系统均支持（需使用 UTF-8 编码）

## ⚠️ 关键规则（Agent 必须遵守，否则交付会被拦截）

1. **Part 2 必须是转录原文**，不是你的总结。P1 才是总结区。写 P2 时必须逐段对照 P3（英文视频）或原始 JSON（中文视频），每翻一段比一段。判别标准：P2 中"因为""所以""但是""而且""可能"等功能连词应 ≥3 个，如果太少说明你替原文做了概括。

2. **永远不要用占位符。** "完整内容请阅读全文""更多内容""后续""待补充"——这些词出现一次，deliver_check.py 就会拦截。

3. **英文视频的 Part 3，原文逐字、加标点、分段。** 不能改写、不能缩写、不能合并段落。只做三件事：加句号问号、按话题分段、修明显 ASR 错误（如 "U.S." → "Just"）。

4. **交付前必须先跑 `deliver_check.py`，通过才能说"完成"。** 没跑 = 没检查 → 没交付。

5. **Part 1 必须是"读完即有收获"，不是"目录式简写"。** 脱离原视频后，读者也能向别人转述具体数字、案例、方法。核心观点部分要用 2-4 句话展开每个论点，不能只罗列标题。

6. **Part 3 英文原文必须合并为自然段落。** YouTube 字幕每 5-8 词换行一次，直接呈现不可读。必须按语义合并为完整句子，保留时间戳 `[MM:SS]` 作为段落开头。示例：
   ```
   ❌ 错误:
   [00:05] I don't know how to teach it
   [00:07] to you except for I hope

   ✅ 正确:
   [00:02] I don't know how to teach it to you except for I hope suffering happens to you.
   ```
   合并规则：连续字幕行属于同一句话时合并，遇到大写字母开头的新句子时另起一段。

7. **音频完整性使用分层校验。** 第1层优先检查 Part 3 时间戳（YouTube 最可靠），第2层用 128kbps 文件大小估算，第3层用转录字数 vs 时长兜底。不要只依赖一层。

8. **中文视频低保留率检测设 35% 硬下限。** 语速 ≥3.5 字/秒时，清理后保留率不得低于 35%（即 `max(35%, 原阈值)`）。低于此值 `deliver_check.py` 将返回错误（exit code 1），Agent 必须升级模型重跑。正常语速（<3.5 字/秒）仍按 45% 阈值执行。

9. **默认使用 tiny 模型快速转录。** 初次转录默认 `--model tiny`。运行 `deliver_check.py` 后，如果因低保留率或转录缺失失败，Agent **必须**先后用 `--model base` 和 `--model small` 重跑完整流程（提取→转写P1/P2/P3→检查），三次均失败才交给用户。用户也可直接要求 `--model small` 或 `--model large-v3` 以跳过自动升级。

10. **中文视频的 Part 2 必须有功能连词（因为/所以/但是/而且/可能）至少 3 个。** 这证明内容是转录原文而非你的总结。口语赘词（我觉得/对吧/就是说/然后/就是）可以安全删除，不算入功能连词。

11. **短链必须先解析。** b23.tv 等短链接必须先用 urllib 拿到 302 跳转后的完整 URL，才能提取 BV 号。不解析的话 B 站视频会获取失败。

## 工作流程

### 笔记三部分标题格式

```
中文视频:
  ## 📋 第一部分：精读稿
  ## 📝 第二部分：中文逐字稿

英文视频:
  ## 📋 第一部分：精读稿
  ## 📝 第二部分：中文逐字稿
  ## 📖 第三部分：英文原文逐字稿
```

> emoji（📋📝📖）是脚本识别分区的标记，但不是必需的。检查脚本同时兼容带 emoji 和不带 emoji 的标题格式：`## 📝 第二部分` 和 `## 第二部分` 都能识别。

### 用户输入

> 用户：发一个视频链接

### Agent 执行

```
Step 1: 检测平台（YouTube/B站/小红书/抖音）
Step 2: 获取数据
  ├── YouTube → youtube_transcript_api API字幕（需代理）
  │               失败时自动回退 Whisper 转录（使用 --model 指定模型）
  ├── B站 → bilibili-api 获取音频 → Whisper（使用 --model 指定模型）
  | 小红书/抖音 → yt-dlp 下载音频 → Whisper（使用 --model 指定模型）
Step 3: 写笔记
  ├── 中文视频:
  │   ├── Part 1: 精读摘要（速读用）
  │   │   ├── 一句话总结（含具体信息）
  │   │   ├── 核心观点（具体数字/案例/方法）
  │   │   ├── 操作建议（如有）
  │   │   └── 适合谁看
  │   └── Part 2: 中文逐字稿（深度阅读用）
  │       ├── 去除口语冗余（"然后""就是""我觉得"等赘词）
  │       ├── 按主题分段 + 加粗小标题
  │       └── 保留全部实质内容，不增删
  └── 英文视频:
      ├── Part 1: 精读摘要（同上）
      ├── Part 2: 中文翻译（全量逐段翻译）
      └── Part 3: 英文原文逐字稿
          ├── 原文字词不改（仅修明显ASR错误）
          ├── 加标点符号
          ├── 按话题分段（3000字以上必须有段落分隔）
          └── 有时间戳（YouTube字幕）
Step 4: 运行 deliver_check.py（21项检查）
   ├── 返回码 0 → 交付给用户
   └── 返回码 1
       ├── 低保留率或转录缺失类错误（[过度清理] / [转录缺失]）
       │   ├── 当前模型 tiny → 用 --model base 重跑 Step 2~4
       │   ├── 当前模型 base → 用 --model small 重跑 Step 2~4
       │   └── 当前模型 small → 仍失败则交给用户（可能是音频质量问题）
       └── 其他类型错误 → 修复具体问题后重跑
```

## 交付前自检（不可跳过）

在每次向用户说"完成"之前，**必须先运行**：
```bash
python scripts/deliver_check.py <笔记文件路径>
```
- 返回码 0 → 可以通过
- 返回码 1 → 必须修复所列问题 → 重新运行 → 直到通过

没有运行 deliver_check.py 之前，不得交付笔记。

## 检查清单（21项，自动执行）

运行 `scripts/deliver_check.py <笔记文件路径>` 自动检查：

### 占位符与质量
| # | 检查项 | 说明 |
|---|--------|------|
| 1 | 无占位符 | 禁止"完整内容请阅读""更多内容""后续"等 |
| 2 | P1四段齐全 | 总结+观点+操作建议+受众 |
| 3 | 核心观点≥150字 | 防止标题堆砌 |
| 4 | 总结≥15字 | 防止空洞 |
| 5 | P2小标题≥2 | 防止无结构 |
| 6 | 功能连词≥3 | 防止重写（因为/所以/但是/而且/可能） |
| 7 | 无重复段落 | 防止内容重复 |
| 8 | P1不超过P2的50% | 防止P2太短 |
| 9 | 文件名合法 | 无非法字符 |
| 10 | Part 3匹配 | 中文无P3 / 英文有P3 |
| 11 | 平台-语言一致 | 抖音/B站/小红书应为中文 |
| 12 | 源数据提醒 | 检查JSON缓存 |

### 内容完整度
| # | 检查项 | 说明 |
|---|--------|------|
| 13 | 音频完整性 | 文件大小 vs 视频时长 ≥85% |
| 14 | 转录完整性 | 字数 vs 时长 ≥最低语速 |
| 15 | Frontmatter格式 | tags用YAML列表 |
| 16 | 过度清理检测 | 中文语速≥3.5字/秒时保留率≥35%（硬下限）；正常语速≥45% |
| 17 | P3完整度 | ≥ raw_chars×65% |
| 18 | 翻译覆盖度 | 中文 ≥ 英文单词×50% |
| 19 | P3标点率 | ≥3‰（每千字符至少3个标点） |
| 20 | P3分段 | 3000字以上必须有换行 |
| 21 | 翻译完整性 | P2小标题 ≥ P3段数×15% |

> ⚠️ 检查不通过不是错误，是质量保障。修完重跑即可通过。

## 常见问题

### Q: 代理怎么配置？
访问 YouTube 需要代理。首次安装时 Agent 会询问你代理地址。常见代理端口：Clash 通常 7897/7890、v2ray 通常 10809/1080。也可在环境变量中设置 `HTTPS_PROXY=http://你的代理地址:端口`。

### Q: 第一次运行要等多久？
首次运行会下载 Whisper tiny 模型（约150MB，远快于 small 的 500MB），视网速需1-5分钟。后续转录：12分钟视频约3-5分钟即可完成。

### Q: 转录质量不够好怎么办？
默认使用 tiny 模型（快速模式）。如果 `deliver_check.py` 报低保留率或转录缺失错误，Agent 会自动用 `--model base` 重跑，仍失败再升级到 `--model small`。你也可以直接使用 `--model small` 或 `--model large-v3` 获得最高精度，也可在检测时加 `--accept-low-retention` 强行通过。

模型速度参考（12分钟视频）：
| 模型 | 耗时 | 质量 |
|:----:|:----:|:----:|
| tiny | ~3-5分钟 | 日常可读 |
| base | ~8-12分钟 | 较好 |
| small | ~15-20分钟 | 高精度 |

### Q: 输出路径不是 Obsidian？
任意文件夹都可以。Agent 会在指定目录下按平台自动创建子文件夹：
```
<你指定的目录>/
├── YouTube/2026-06/
├── B站/2026-06/
├── 小红书/2026-06/
└── 抖音/2026-06/
```

### Q: 为什么要有这么多检查？
因为 Agent 会犯三大错误：①用占位符替代内容 ②在 Part 2 写自己的总结而非原文 ③英文 Part 3 不完整/无标点/无分段。这21道检查是经验教训的总结，防止重复踩坑。

### Q: 检查被拦住6-7项，正常吗？
正常。修一项跑一次脚本就行，不需要一次全修完。脚本会告诉你具体哪里有问题。

### Q: macOS/Linux 会不会报错？
理论兼容。已知差异：ffmpeg 需要自行安装（`brew install ffmpeg` 或 `apt install ffmpeg`）。路径格式 Python 自动处理。如果遇到问题请提 GitHub Issue。
